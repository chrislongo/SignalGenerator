## Audio Engine

All audio synthesis is consolidated in `AudioEngine.swift` — a single `AVAudioSourceNode` render callback handles all waveform and noise types.

### Signal Chain

```
AVAudioSourceNode (render callback @ 48 kHz)
    → AVAudioMixerNode → AVAudioOutputNode (speakers)
```

### PolyBLEP Anti-Aliasing

Square and sawtooth waves use PolyBLEP (Polynomal Band-Limited Step) to smooth discontinuities at waveform transitions, eliminating aliasing artifacts at high frequencies. Triangle waves are generated by integrating a PolyBLEP'd square wave, producing a naturally band-limited result.

### Render Callback

The callback runs on a real-time audio thread. Rules:
- **No allocations** — no `Array`, `String`, or any heap allocation
- **No locks** — no `DispatchSemaphore`, `NSLock`, `os_unfair_lock`
- **No Objective-C** — no message sends, no `@objc` calls

Parameters are bridged from the main thread via `AudioParameters`, a class using naturally-aligned 32-bit loads/stores (`Float` and `Int32`) for lock-free thread safety on ARM64.

### Waveform Math

All waveforms computed from a normalized phase `p` in `[0, 1)`:

| Waveform | Formula |
|----------|---------|
| Sine | `sin(2π * p)` |
| Square | `p < 0.5 ? 1.0 : -1.0` |
| Sawtooth | `2.0 * p - 1.0` |
| Triangle | `4.0 * abs(p - 0.5) - 1.0` |
| White noise | LCG-based random per sample |
| Pink noise | Voss-McCartney algorithm |

### Seamless Waveform Switching

A 256-sample (~5ms) crossfade is applied when switching waveform types to prevent audible pops. The new waveform fades in over the crossfade window.

### Volume

Volume is applied as a linear multiplier in the render callback (`sample * volume`). Volume steps: 0% to 100% in 10% increments (11 levels). Default: 50%.

## State Management

### SignalState (@Observable)

Single source of truth, observed by all views:

```swift
@Observable
class SignalState {
    var frequency: Double = 440.0      // 10...20000
    var waveform: WaveformType = .sine
    var stepIncrement: Double = 1.0     // 0.1, 1, 10, 100
    var volume: Int = 5                 // 0...10 (maps to 0%...100%)
    var isPlaying: Bool = true
}
```

Uses Swift 5.9 `@Observable` macro (iOS 17+). Note: do not use `didSet` to clamp values on `@Observable` properties — it causes re-entrant observation issues. Clamping is done at call sites instead.

### Audio Thread Communication

`ContentView` propagates state changes to the audio engine via `.onChange` modifiers that call `audioEngine.update(state:)`, which writes to the `AudioParameters` bridge.

## Jogwheel

### Gesture Handling

The jogwheel uses a `DragGesture` mapped to angular rotation:

1. On drag start: record initial touch angle relative to wheel center
2. On drag change: compute angular delta, apply frequency change scaled by step increment
3. On drag end: stop (no inertia)

### Frequency Snapping

When the step increment is >= 1 Hz, frequency snaps to the nearest multiple of the step size. This applies both during jogwheel rotation and when changing the step increment. For example, with step 10: 440 → 450 → 460.

### Visual Rotation

The wheel face rotates via `.rotationEffect()`. The thumb indent rotates with it since it's part of the same view.

## Display Rendering

### Waveform Visualization (Canvas)

SwiftUI `Canvas` view with `TimelineView(.animation)` drives 60fps updates. The waveform draws a `Path` based on the same math as the audio engine but for display purposes only. Number of visible cycles scales: `clamp(frequency / 80, 1.5, 12)`.

### Direct Frequency Input

Tapping the frequency readout in the display opens `FrequencyInputView` — a calculator-style numeric keypad that replaces the waveform and readout. Users type a value and confirm with **Hz** (direct) or **kHz** (×1000). ESC cancels. Values outside the 10–20000 Hz range are rejected.

### CRT Effects

All done with SwiftUI modifiers, no shaders:
- **Scanlines**: Canvas-drawn horizontal lines
- **Screen curvature highlight**: Radial gradient positioned top-left
- **Grid**: Drawn in the Canvas as thin lines
- **Bezel**: `RoundedRectangle` with dark border
- **Glow on waveform**: Double-draw technique (thick transparent pass + thin opaque pass)

## Component: ChunkyButton

Implemented as a custom `ButtonStyle` applied via `.chunkyButtonStyle(isActive:)`. Active buttons appear pressed in (offset down, no shadow). Inactive buttons have a 3D raised appearance with bottom shadow.

## Theme

Centralized in `Theme.swift` with a `Color(hex:)` extension. Fonts accessed via static methods: `Theme.mono()`, `Theme.monoBold()`, `Theme.display()`, `Theme.displayHeavy()`.

### Color Palette

- Chassis: dark grey (#2a2a2c)
- Display: dark screen with teal (#48c8d8) waveform, amber (#f0982c) readout
- Header: teal (#38a8a0)
- Buttons inactive: medium grey (#484848)
- Buttons active: off-white/cream (#d8d0c8) with dark text
- Power button: red (#d03828) when active
